Projects I'm considering

An RL & optimization sandbox

Three connected experiments built around one idea: how do you get an agent to find a good policy on its own — and how do you know when it actually has? The throughline is measurement discipline. Most of the hard-won lessons here are about not fooling yourself with the metric.

Projects

Chrome Dino RL

Validated dino_rl/

Agents that teach themselves the Chrome offline dinosaur game. A Double/Dueling DQN and a genetic algorithm, both trained in a fast Python simulation that mirrors the browser game constant-for-constant, then validated playing the real thing.

Result: both agents complete a 4-phase curriculum with zero human intervention and reach the same near-perfect ceiling (eval 11,087). The genetic version does it in a 419-parameter genome. Training runs ~1000× faster than the browser (~35k steps/sec vs ~35).

Factory Sim

Working factory_sim/

A manufacturing-line simulator. Define a production line in YAML, then let three agents — greedy ROI, random search, and a DQN — compete to find the best capital- investment plan within a budget. Generates HTML reports of the economics.

The hook: the showcase scenario's optimum is counter-intuitive — fix the bottleneck's yield first, then add capacity. A greedy agent chases the immediate capacity gain and burns money running a high-scrap process faster.

OptiFlow

Working factory_chat/

A chat interface over the factory simulator. Talk to it like a plant analyst — "where's my bottleneck?", "what if we add a second CNC machine?" — and it runs the sim, builds named investment scenarios, tracks in-flight commitments, and compares plans side by side.

Core design: the plant model is the source of truth for physical reality; scenarios capture hypotheticals. You branch a named scenario rather than editing reality — which keeps every number auditable.

What I learned

Train in a sim, validate in the real thing

A deterministic Python mirror removed the browser's ~10–20ms action jitter and ran ~1000× faster. Feature parity meant a sim-trained network played the real browser game flawlessly — verified, not assumed.

Control decisions must key off a clean measurement

Training scores come from the exploring policy — "policy + noise." A greedy, fixed-seed eval exam drives every gate, checkpoint, and stall decision instead. The noisy score is display-only.

Shape difficulty through the environment, not the reward

Rewards stay sparse and stationary; difficulty ramps via the environment (speed, obstacles). Changing the reward mid-run poisons the replay buffer and moves the target out from under what's already learned.

When a learner plateaus, check the optimizer's signal first

The genetic algorithm looked like it topped out far below the DQN. It hadn't — a fixed fitness cap had saturated selection so it couldn't rank survivors. An adaptive cap fixed it and evolution matched the DQN exactly. The apparent capacity gap was a measurement bug.
